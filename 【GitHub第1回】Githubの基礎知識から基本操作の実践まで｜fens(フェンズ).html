【GitHub第1回】Githubの基礎知識から基本操作の実践まで

GitHubとは

一言でいうと「プログラムソースの管理ツール。」このツールを使えばプログラムコードをクラウドで保存しながらバージョン管理も出来る。

Publicに公開されているものはオープンソース

Githubにはプログラムのソースコードをクラウドに上げた際の状態として2パターン存在する。

  • Public　・・・全世界、誰でもコードの閲覧、ダウンロードが出来る

  • Private　・・・作成者から招待された人のみ閲覧、ダウンロード出来る

Publicのコードは実質オープンソース化してあり、コードの商用利用や著作権違反を制限することは不可能。その反面、利用者とGitHub上でディスカッション出来ることもあるので自身のスキルアップを見込める可能性もある。

Privateに公開し、限定的なコミュニティで共有する方法もある

一方Privateでアップし、アップロード者が許可した者のみアクセスできるようにすることも出来る。
これは機密情報や一般には知られたくないデータ、学習済みAIモデルをアップしている場合には有効と言える。
（※そもそも機密情報をオープンなクラウドにアップする是非について議論の余地はありそうだが）

基本的な使い方はチームエンジニアリング

GitHubはチームエンジニアリングする際に便利である。
AチームがA'の機能を作りこみ、BチームがB'の機能を作りこむ。
こういう場合GitHubがあれば同時にタスクをこなせ、容易に結合できるが
GitHubが無かったらA'とB'のソースコードを慎重に結合していかなければならない。

またA'の機能が無いバージョンでリリースする。という決定がなされた場合にも容易に実現できる。
GitHubがなかったらA'のコードはどこにあたるのか、さかのぼって確認する必要があるが、GitHubがあるとcheckpointが記録されているので簡単に戻すことが出来る。

webアプリを簡単に試せるHuggingFaceというものも存在する

ここでは紹介だけにとどめておくが、webアプリを簡単に試せるHuggingFaceというものも存在する。イメージ的には「GUIも使えるGitHub」といった感じに近い。
もし気になる声が上がればHuggingFaceについても情報展開しようと思う。

GitHubの基本用語

GitHubを扱うにあたって基本的な用語は押さえておく必要がある。この用語を理解しておかないと何が何だか分からない状態で操作してしまい、大惨事となることもあり得る。

状態編

  • repository（リポジトリ）・・・プログラムの保存場所。PCのフォルダの考えに近い。

  • master（マスター）・・・大元のプログラム。マスターが書き換わるとリリース時の挙動も変わるので注意が必要。

  • branch（ブランチ）・・・マスターから追加機能作成、修正時に作成する。ブランチ作成後、ローカルPCに引き込み、機能部分を作成したりする。挙動が上手く行けばリモートにアップし、マスターと結合（マージ）する。

  • checkpoint（チェックポイント）・・・ブランチからマスターに結合された時点のこと。機能削除や不具合調査時に”この時点まで戻したい”といったときに役立つ。

操作編

  • pull（プル）・・・リモートリポジトリからローカルPCにプログラムを引き込むこと。チームメンバーでプログラム修正している際に役立つ。

  • commit（コミット）・・・ローカルPCの変更をプル時点のプログラムに反映すること。この時点でバグが発生しないか確認することが出来る。

  • push（プッシュ）・・・リモートリポジトリのブランチに変更を反映すること。pushを行えば作成途中のコードを別の人に受け渡すことも簡単にできる。

  • merge（マージ）・・・ブランチで変更した部分をマスターに反映すること。push後の機能が問題なく動いていることを確認出来たら行う。

簡単な絵にまとめてみるとこんな感じ




GitHubの使い方

0.VisualStudio Codeとの相性がいい

まず理解して欲しいのは、複雑なプログラムを作ろうとすれば必ず"Visual Studio Code"が必要になる。
Google Colabを使えば簡単にPythonの実行、AI学習させることが出来るが、
最終的にプログラムを使うのであれば最初からVisualStudioを使うのも手段としてはアリ。
VisualStudioにあって、Google Colabにはない物、、、それは”GitHub”との互換性である。VisualStudioの方が圧倒的にGitHubの操作が簡単にできる。
通常GitHubを使いたい場合"git ~~"から始まるコマンドを覚える必要があるが、VisualStudioを使えば覚える必要が無い。クリックなどの感覚的な操作でGitHubを扱うことが出来る。

ここでは手始めとしてVisualStudio Codeを使ったGitHubの操作を経験してもらう。

まとめ

  • GitHubはソース管理、チームエンジニアリングには便利なツール

  • 多くのAIがGitHubで公開されており、自由に使うことが出来る。（本よりも情報が豊富）

  • GitHubには独自の用語、操作、考え方があり、慣れるまでは複雑

  • Python開発を行う上で十中八九通る道なので、知っていて損はない
